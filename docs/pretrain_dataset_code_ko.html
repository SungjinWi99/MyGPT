사전학습 Dataset 코드 설명

이번에 만든 PackedPretrainDataset은 우리가 생성한 pretrain/v*/parquet/{train,validation} 데이터를 바로 학습에 쓰기보다, 한 번 토큰화해서 고정 길이 binary cache로 저장한 뒤 재사용하는 Dataset이다.

왜 KoAlpacaDataset을 버렸나

기존 KoAlpacaDataset은 instruction, output, url 같은 SFT 데이터 형식을 전제로 했다. 지금 우리가 만든 데이터셋은 사전학습용 일반 한국어 원문 Parquet이고, 핵심 컬럼은 text다. 따라서 문서 텍스트를 이어 붙여 causal language modeling 샘플로 만드는 Dataset이 필요하다.

구분 기존 방식 새 방식
목적 지시-응답 SFT 일반 한국어 말뭉치 사전학습
입력 구조 instruction, output parquet/{split}/*.parquet의 text
label 정책 프롬프트 구간은 -100 masking 전체 토큰에 대해 next-token prediction
성능 전략 매 샘플을 즉시 토큰화 한 번 packed binary cache를 만들고 재사용

전체 흐름

1
Parquet split 찾기

dataset_dir/parquet/train/*.parquet 또는 dataset_dir/parquet/validation/*.parquet를 읽는다.

2
text 컬럼만 batch 단위로 읽기

전체 파일을 메모리에 올리지 않고 pyarrow.parquet.ParquetFile.iter_batches로 streaming 처리한다.

3
tokenizer로 토큰화

add_special_tokens=False로 원문 자체를 토큰화하고, 문서 끝에는 eos_token_id를 하나 붙인다.

4
max_seq_len + 1 토큰 샘플 만들기

학습 입력은 앞 max_seq_len개, label은 한 칸 뒤로 민 max_seq_len개가 필요하므로 저장 단위는 항상 max_seq_len + 1이다.

5
binary cache 저장 후 memmap으로 읽기

uint32 binary 파일과 metadata JSON을 만들고, 학습 중에는 np.memmap으로 필요한 row만 읽는다.

Dataset 출력 형태

__getitem__은 하나의 packed token row를 가져와서 다음처럼 나눈다. 

tokens = [t0, t1, t2, ..., t1024]  # max_seq_len=1024라면 총 1025개

input_ids = [t0, t1, t2, ..., t1023]
labels    = [t1, t2, t3, ..., t1024]

그래서 학습 loop에서는 logits와 labels를 별도로 한 칸 더 shift할 필요가 없다. 모델 출력 [B, T, vocab]와 labels [B, T]를 그대로 cross entropy에 넣는 구조가 자연스럽다.

주요 인자

dataset_dir

데이터셋 버전 루트다. 예: /content/drive/MyDrive/KTB/MyGPT/datasets/pretrain/v2

max_seq_len

모델 context length와 맞아야 한다. 현재 설정이면 보통 1024. cache도 이 값에 종속된다.

split

train 또는 validation. 각 split은 별도 cache 파일을 가진다.

token_budget

smoke, pilot, main run에서 학습 토큰 수를 제한할 때 쓴다. 예를 들어 100_000_000이면 약 1억 training token까지만 cache를 만든다.

rebuild_cache

이미 만들어진 cache를 무시하고 다시 만들 때만 True로 둔다. tokenizer나 sequence length를 바꾸면 새 cache가 필요하다.

cache_dir

기본값은 dataset_dir/tokenized/packed-<tokenizer>-seq<max_seq_len>. 실험별 cache를 따로 두고 싶을 때 직접 지정할 수 있다.

Colab 사용 예시

from transformers import PreTrainedTokenizerFast
from torch.utils.data import DataLoader

from src.data import PackedPretrainDataset, collate_fn

tokenizer = PreTrainedTokenizerFast.from_pretrained(
    "skt/kogpt2-base-v2",
    bos_token="</s>",
    eos_token="</s>",
    unk_token="<unk>",
    pad_token="<pad>",
    mask_token="<mask>",
)

dataset = PackedPretrainDataset(
    "/content/drive/MyDrive/KTB/MyGPT/datasets/pretrain/v2",
    tokenizer,
    max_seq_len=1024,
    split="train",
    token_budget=100_000_000,
)

loader = DataLoader(
    dataset,
    batch_size=16,
    shuffle=True,
    num_workers=2,
    pin_memory=True,
    collate_fn=collate_fn,
)

cache 파일 구조

기본 설정이면 다음 위치에 cache가 생긴다. 

pretrain/v2/
  parquet/
    train/
    validation/
  tokenized/
    packed-skt-kogpt2-base-v2-seq1024/
      train.uint32.bin
      train.meta.json
      validation.uint32.bin
      validation.meta.json

*.uint32.bin에는 토큰 id가 고정 길이 row로 저장되고, *.meta.json에는 sequence length, sample 수, training token 수, EOS id, packing policy가 기록된다.

주의할 점

첫 실행은 Parquet 전체를 읽고 tokenizer를 돌리기 때문에 오래 걸릴 수 있다. 하지만 한 번 cache가 만들어지면 이후 Dataset 초기화는 binary 파일을 memmap으로 여는 정도라 훨씬 빠르다.

tokenizer, max_seq_len, packing 정책, 데이터셋 버전이 바뀌면 기존 cache를 재사용하면 안 된다. 이 경우 새 cache_dir를 쓰거나 rebuild_cache=True로 다시 만들어야 한다.

현재 src.train은 이 Dataset을 사용하는 사전학습 루프로 연결되어 있다. loss 계산은 이미 shift된 labels 기준으로 수행하며, validation, token-based logging, W&B sample table, step checkpoint, resume을 지원한다.
