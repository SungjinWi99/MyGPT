교육 목적의 첫 사전학습 기준선을 현대적인 소형 LLM 구조로 다시 구현하기 위한 설계 메모다. 각 개념의 입문 설명보다 코드에 반영할 선택과 검증 포인트를 중심으로 정리한다.

권장 기준선

기존 4-layer 구조를 유지하지 않고, 아래 8-layer 구조를 첫 공식 기준선으로 사용한다. 다만 각 요소는 독립 모듈로 구현해 이후 ablation이 가능해야 한다. 
model:
  d_model: 512
  n_layers: 8
  n_heads: 8
  head_dim: 64
  max_seq_len: 1024
  position_encoding: rope
  normalization: rmsnorm
  ffn_type: swiglu
  ffn_hidden_size: 1408
  tie_embeddings: true
  attention_backend: sdpa
  dropout: 0.0
  use_bias: false

목표는 모델 크기를 무작정 늘리는 것이 아니라, 중복된 출력 임베딩 파라미터를 Transformer 블록의 깊이와 표현력에 재배분하는 것이다.

개선 요소

1. Weight Tying

토큰 임베딩 행렬 E를 출력 projection에도 사용한다. 별도 Linear(d_model, vocab_size)를 만들지 않는다. 

self.token_embedding = nn.Embedding(vocab_size, d_model)

logits = F.linear(hidden_states, self.token_embedding.weight)

초기화와 checkpoint state dict에서 동일 tensor가 한 번만 관리되는지 확인한다.

2. 8-layer 깊이

d_model=512와 8 heads는 유지하고 블록만 8개로 늘린다. 각 블록은 다음 pre-norm residual 형태를 사용한다. 

x = x + attention(rmsnorm_attn(x))
x = x + swiglu(rmsnorm_ffn(x))

두 normalization 모듈은 공유하지 않는다. 마지막 logits 이전에도 별도 final RMSNorm을 적용한다.

3. RoPE

임베딩에 위치 벡터를 더하지 않고, head로 reshape한 Q와 K에 회전을 적용한다. 

q = apply_rope(q, cos[:seq_len], sin[:seq_len])
k = apply_rope(k, cos[:seq_len], sin[:seq_len])
  • head_dim=64는 짝수인지 검증한다.
  • cos/sin은 persistent buffer가 아닌 캐시로 둔다.
  • 기준선의 rope_theta는 10000으로 둔다.
  • 1024 context에서는 scaling 기법을 적용하지 않는다.

4. RMSNorm

rms = x.pow(2).mean(dim=-1, keepdim=True)
x_norm = x * torch.rsqrt(rms + eps)
return weight * x_norm
  • 학습 파라미터는 element-wise scale 하나만 둔다.
  • eps=1e-6을 기준값으로 사용한다.
  • BF16에서도 제곱 평균은 필요하면 FP32로 계산한다.

5. SwiGLU

gate, value = input_projection(x).chunk(2, dim=-1)
hidden = F.silu(gate) * value
output = output_projection(hidden)

기존 4 × d_model = 2048 GELU FFN과 계산량을 비슷하게 맞추기 위해 hidden size를 약 8d/3로 잡는다. d=512에서는 텐서 코어 친화적인 1408을 권장한다.

6. SDPA

output = F.scaled_dot_product_attention(
    q,
    k,
    v,
    attn_mask=None,
    dropout_p=0.0,
    is_causal=True,
)
  • 별도 하삼각 attention mask buffer를 제거한다.
  • Q/K/V shape는 [B, heads, T, head_dim]으로 맞춘다.
  • 학습 중 attention weight 자체는 반환하지 않는다.
  • CPU 테스트와 CUDA 결과의 허용 오차를 따로 둔다.

Bias와 초기화

  • Q/K/V/O projection과 SwiGLU projection은 bias=False.
  • Embedding과 linear weight는 우선 std=0.02 정규분포로 초기화.
  • Residual branch 출력 projection은 깊이에 맞춘 scaled initialization 검토.
  • 첫 기준선에서는 복잡한 μP나 DeepNorm 초기화는 도입하지 않는다.

파라미터 배분

KoGPT2 tokenizer의 약 51,200 vocab과 d_model=512를 기준으로 한 근사치다.

구성 근사 파라미터 비고
토큰 임베딩 26.2M 입출력 공유
Attention / layer 약 1.05M Q, K, V, O projection
SwiGLU / layer 약 2.16M 512 → 2×1408 → 512
8 Transformer blocks 약 25.7M Norm 파라미터 포함 근사치
전체 약 52M 현재 약 65M보다 작고 블록은 두 배
파라미터 수 감소가 곧 계산량 감소를 뜻하지는 않는다. 4층에서 8층으로 늘어나므로 token당 block 연산량은 증가한다. A100에서의 실제 tokens/sec는 500만 토큰 스모크에서 측정해 1억·5억 토큰 실행 시간을 다시 산정한다.

권장 구현 순서

  1. Config와 shape contract
    새 옵션을 dataclass에 추가하고 d_model % n_heads == 0, RoPE 차원, context 길이를 검증한다.
  2. RMSNorm 단위 구현
    FP32 reference 식과 forward/backward를 비교한다.
  3. RoPE 단위 구현
    cos/sin cache, even/odd pair 회전, device와 dtype 이동을 검증한다.
  4. SwiGLU 구현
    projection shape와 hidden size를 독립 테스트한다.
  5. SDPA attention 구현
    작은 tensor에서 기존 manual causal attention과 출력을 비교한다.
  6. TransformerBlock 조립
    pre-norm residual 순서와 gradient 흐름을 확인한다.
  7. Weight tying 적용
    logits shape, 동일 storage, 저장·복원 후 공유 상태를 검증한다.
  8. 전체 모델 smoke test
    random token batch로 forward, cross entropy, backward, optimizer step을 실행한다.
  9. 사전학습 trainer 연결
    1,024-token packed dataset, BF16, token budget, validation, checkpoint/resume, W&B를 연결한다.

필수 검증 항목

단위 테스트

  • 각 모듈의 입력·출력 shape
  • causal leakage 없음
  • RoPE cache의 dtype/device 일치
  • RMSNorm reference 일치
  • SDPA와 manual attention 수치 비교
  • embedding weight storage 공유

학습 테스트

  • 작은 batch에서 loss 감소
  • gradient NaN/Inf 검사
  • BF16과 FP32 loss 차이 확인
  • checkpoint 전후 logits 일치
  • resume 후 token/step 연속성
  • 실측 peak GPU memory와 tokens/sec

완료 기준

  • 모델 파라미터 수가 자동 계산되어 W&B config에 기록된다.
  • 고정 seed의 CPU forward test가 재현된다.
  • A100 BF16에서 1,024-token batch의 forward/backward가 안정적으로 돈다.
  • 5백만 토큰 스모크 동안 loss가 지속적으로 하락하고 NaN이 없다.
  • 중간 checkpoint에서 재개한 결과가 token budget을 정확히 이어간다.

후속 비교 실험

첫 기준선을 완성한 뒤 한 번에 한 요소만 바꾸어 비교한다. 모든 비교는 같은 packed-data fingerprint, token budget, seed, optimizer 설정을 사용한다.

우선순위 비교 관찰값
1 4 layers vs 8 layers 동일 토큰·시간에서 validation loss와 tokens/sec
2 GELU vs SwiGLU parameter/FLOP 보정 후 loss
3 Sinusoidal vs RoPE 긴 prompt 생성 안정성과 validation loss
4 LayerNorm vs RMSNorm 학습 안정성, 속도, loss
5 weight tying on/off 같은 token budget에서 perplexity와 메모리
6 dropout 0.0 vs 0.1 과적합 징후가 있을 때만 수행
